## рҹҸ—пёҸ Open-Source Simulation Foundation

> **All test case driving scenarios in ENIGMAA are simulated using the open-source [`highway-env`](https://github.com/eleurent/highway-env) library.**

### What is highway-env?

[`highway-env`](https://github.com/eleurent/highway-env) is a **free, open-source collection of environments for autonomous driving and tactical decision-making**, originally developed by **Гүdouard Leurent** and now maintained by the **[Farama Foundation](https://farama.org)** as part of the Gymnasium ecosystem.

It provides:
- Kinematic vehicle simulation with realistic lane/road geometry
- Multi-agent NPC traffic with configurable spawn rates and behaviors
- Observation spaces: kinematics, occupancy grids, grayscale frames
- Action spaces: discrete (lane change/speed) and continuous (steering/acceleration)
- Multiple pre-built environments covering highway, intersection, roundabout, parking, and racetrack scenarios

**Our modifications** on top of the base library:

```
highway-env (upstream, MIT)
        в”Ӯ
        в”ңв”Җв”Җ patch_intersection.py      вҶ’ intersection-v0 AEB logic + action remapping
        в”ңв”Җв”Җ patch_coordinates.py       вҶ’ relative ego-coordinate trajectory encoding
        в”ңв”Җв”Җ patch_data_architecture.py вҶ’ per-scenario isolated .npz data files
        в””в”Җв”Җ patch_files.py             вҶ’ FileLock-safe concurrent training support
```

**MC Dropout** remains active during inference. Each forward pass samples a different trajectory distribution, and N passes are aggregated to estimate:

- **Epistemic uncertainty** вҖ” what the model *doesn't know* (reducible with more data)
- **Aleatoric uncertainty** вҖ” inherent observation noise (irreducible)

### MPC Action Selection Loop

## рҹӣЎпёҸ Hard Shield вҖ” AEB Safety System

The **Hard Shield** is a **physics-level, rule-based safety override** that executes *after* the neural network selects an action. It acts as a non-negotiable last-resort collision barrier вҖ” the neural policy is never solely responsible for collision avoidance.

### Highway / Two-Way Mode (Lane-Based Geometry)

```
  Ego Vehicle в”Җв”Җв–¶ Check surrounding vehicles:

  в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
  в”Ӯ  crash_ahead   (|dy| < 2.0m,  0 < dx < 15.0m)  вҶ’  FRONT   в”Ӯ
  в”Ӯ  blocked_left  (-6.0m < dy вүӨ -2.0m, |dx| < 10m) вҶ’  LEFT   в”Ӯ
  в”Ӯ  blocked_right  (2.0m вүӨ dy < 6.0m,  |dx| < 10m) вҶ’  RIGHT  в”Ӯ
  в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

  Decision Tree:
  crash_ahead + action вҲҲ {IDLE, FASTER}
      в”ңв”Җв”Җ NOT blocked_left  в”Җв”Җв–¶  action = LANE_LEFT   (evade)
      в”ңв”Җв”Җ NOT blocked_right в”Җв”Җв–¶  action = LANE_RIGHT  (evade)
      в””в”Җв”Җ BOTH blocked      в”Җв”Җв–¶  action = SLOWER       (brake)

  action = LANE_LEFT  + blocked_left  в”Җв”Җв–¶  IDLE (or BRAKE if crash)
  action = LANE_RIGHT + blocked_right в”Җв”Җв–¶  IDLE (or BRAKE if crash)
```

### Intersection Mode (Radial Proximity)

```
  For every NPC vehicle v:
    dist = sqrt[(v.x вҲ’ ego.x)ВІ + (v.y вҲ’ ego.y)ВІ]
    dist < 8.0m  в”Җв”Җв–¶  SLAM BRAKES  рҹӣ‘  (action = SLOWER)

  Action remapping (NN 5-class вҶ’ intersection-v0 Discrete(3)):
    NN:  LEFT=0  IDLE=1  RIGHT=2  FASTER=3  SLOWER=4
    Env:   вҶ’1      вҶ’1      вҶ’1       вҶ’2         вҶ’0
```

## рҹ”§ Patch System

```

        в””в”Җв”Җ patch_intersection.py       вҶ’ Switch to intersection-v0
```

## рҹ”¬ Explainability (XAI)

ENIGMAA uses **[Captum](https://captum.ai/) Integrated Gradients** to trace every action decision back to the raw 69-dimensional input features вҖ” fulfilling safety and audit requirements in regulated deployments.

```
ModelWrapper(BayesianTrajectoryPredictor)
        в”Ӯ
        в””в”Җв”Җ IntegratedGradients.attribute(input_tensor)
                в”Ӯ
                в–ј
        Attribution scores per input feature
        (69 values: 64 kinematic dims + 5 action dims)
                в”Ӯ
                в–ј
        Published to /explainability/rationale  (ROS2)
        or logged inline in terminal output
```

The `ModelWrapper` reduces the multi-modal GMM output (3 modes Г— 30 steps Г— 2 coords) to a scalar score so that Integrated Gradients can compute gradients through the full prediction stack.

> This enables safety engineers and regulators to audit *why* the agent selected a particular action at any timestep вҖ” a key requirement for ISO 26262 / SOTIF compliance in production AV systems.

## вҡҷпёҸ Configuration Reference

All environments are configured by passing a `config` dict to `gym.make()`:

```python
env = gym.make("highway-v0", render_mode="human", config={
    "lanes_count": 2,
    "duration": 1_000_000,
    "simulation_frequency": 15,
    "collision_reward": -50,
    "high_speed_reward": 1,
    "right_lane_reward": 0.1,
    "reward_speed_range": [20, 30],
})
```

| Parameter | Type | Default | Description |
|---|---|---|---|
| `lanes_count` | `int` | `4` | Number of highway lanes |
| `duration` | `int` | `40` | Episode length in steps |
| `simulation_frequency` | `int` | `15` | Simulation steps per second |
| `collision_reward` | `float` | `-1` | Penalty applied on collision |
| `high_speed_reward` | `float` | `0.4` | Reward for high-speed travel |
| `right_lane_reward` | `float` | `0.1` | Reward for right-lane preference |
| `reward_speed_range` | `list` | `[20, 30]` | Speed range for full reward |
| `spawn_probability` | `float` | `0.6` | NPC vehicle spawn rate (intersection) |
| `arrived_reward` | `float` | `1.0` | Reward for completing intersection cross |
| `manual_control` | `bool` | `False` | Enable keyboard control |
